TinyRouter UI/UX 现代化改造 — 协助请求文档

发给高级 UI 设计 agent 的请求附件：无需阅读项目源码，即可理解当前视觉 / 交互定义、约束与改造目标。
TinyRouter 是单二进制 LLM API 代理 + 本地工具集，内置原生 HTML + vanilla JS + CSS 的 Web 管理界面（通过 Go embed.FS 内嵌）。所有 DOM 由 JS 在运行时动态生成。本次改造仅针对 CSS + HTML 结构现代化，不动 JS 业务逻辑。

第 1 节 · 改造约束 最高优先级

先于一切

以下约束为本项目改造的硬性边界。任何设计提案若违反任一约束，均不可接受。

约束说明
改造范围主应用（Usage / Settings / Console / Download + 主题 / 导航 / 通用控件）+ Playground 两套前端全部整改
可改动层仅 CSS + HTML 结构（两个 index.html 静态结构）。不得改动 JS（业务逻辑、API 调用、SSE、状态管理、动态 DOM 生成逻辑完全不动）。必要时可调整 JS 中字符串拼接 HTML 模板里的内嵌 style/class（这是 HTML 输出层不是逻辑层）——但按本附件授权，仅授权 CSS 改动。
浏览器兼容现代浏览器（2024+）：允许使用 :has()、container queries、subgrid、color-mix() 等新特性
主题必须保留深色 + 浅色双主题；现行机制是 :root + [data-theme="light"] 切换 CSS 变量，禁改切换机制
字号保留 data-font-size S / M / L 三档切换机制
语言保留 data-lang en / cn 切换机制
版本管理git 分支直接覆盖原文件，不留废弃文件，回滚用 git revert
不得改动Z:\Playground\9router 不可动；引入第三方依赖；数据库
布局原则不破坏原布局骨架：动作（点击 / 拖拽 / SSE）+ 元素层级 + 信息架构保留
响应式当前断点 768px / 480px；可优化但保持信息密度
第三方组件xterm.js 容器、Chart.js canvas、KaTeX、Mermaid、highlight.js 保持渲染层不动，可改包裹容器样式
